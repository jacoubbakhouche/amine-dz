# ⚡ CORS Fix - Quick Deploy

## 3 خطوات فقط لحل CORS Error:

### ✅ الخطوة 1: تحديث الـ Code (تم بالفعل!)
الملف تم تعديله: `supabase/functions/pro-rag-consultation/index.ts`

**ماذا تم:**
- ✅ إضافة CORS_HEADERS config
- ✅ معالجة OPTIONS requests
- ✅ إضافة CORS headers لجميع responses
- ✅ رفض requests غير الـ POST

### ✅ الخطوة 2: Deploy الـ Function الجديدة

```bash
# في Terminal:
supabase functions deploy pro-rag-consultation --project-id YOUR_PROJECT_ID
```

**التحقق من النجاح:**
```
✓ Function deployed successfully
✓ Endpoint: https://YOUR_PROJECT.supabase.co/functions/v1/pro-rag-consultation
```

### ✅ الخطوة 3: Hard Refresh + Test

```
1. افتح المتصفح
```

## ❌ إذا لم ينجح:

### 1. تأكد من الـ Deploy
```bash
# تحقق من الـ deployment
supabase functions list --project-id YOUR_PROJECT_ID
```
يجب أن ترى: `pro-rag-consultation` في القائمة

### 2. حاول Redeploy
```bash
supabase functions deploy pro-rag-consultation \
  --project-id YOUR_PROJECT_ID \
  --force
```

### 3. تحقق من الـ Logs
```bash
supabase functions logs pro-rag-consultation
```
يجب أن تراه: `[CORS] Handling OPTIONS preflight request`

### 4. Clear Browser Cache
- Chrome: Cmd+Shift+Delete (Mac) / Ctrl+Shift+Delete (Windows)
- ثم: اختر "All time"
- اضغط: "Clear data"

## 📊 CORS Headers الجديدة

```typescript
{
  "Access-Control-Allow-Origin": "*",           // ✅ من أي origin
  "Access-Control-Allow-Methods": "POST, OPTIONS, GET",
  "Access-Control-Allow-Headers": "Content-Type, Authorization, apikey",
  "Access-Control-Max-Age": "86400",            // ✅ Cache 24 ساعة
  "Access-Control-Allow-Credentials": "true",
}
```
